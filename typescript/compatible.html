# 类型兼容性

类型兼容性是基于结构子类型的。

# 兼容性特点

interface ISchool {
    name: string,
    age: number
}
let fn = (obj: ISchool): string => {
    return obj.name;
}
fn({
    name: '珠峰',
    age: 10,
    address: '回龙观东大街' // 对象文字可以只指定已知属性，并且“address”不在类型“ISchool”中
});

# 类兼容性

两个类类型的对象时，只有实例的成员会被比较

class Animal {
    static type = '哺乳类';
    public maxAge = 500;
}
class Person {
    public maxAge = 100;
}
let person: Person = new Animal();

类的私有成员和受保护成员会影响兼容性,如果目标类型包含一个私有成员，那么源类型必须包含来自同一个类的这个私有成员

class Animal {
    static type = '哺乳类';
    private maxAge = 500;
}
class Person { // 可以让person类继承Animal的类
    private maxAge = 100;
}
let person: Animal = new Person(); // 不能将类型“Person”分配给类型“Animal"
